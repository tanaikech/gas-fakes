# v2.2.3 Release Notes - Microsoft Graph Refinements & Naming Standardization

This release focuses on hardening the Microsoft Graph (OneDrive) authentication flow and standardizing terminology across the codebase and documentation.

## What's Changed?

### 1. Platform Naming Standardization
- Standardized the platform identifier from `workspace` to `google` everywhere.
- Updated `ScriptApp.__platform` and `ScriptApp.__platformAuth` to use `google` as the standard backend name.
- Updated all documentation and examples to match this convention.

### 2. Microsoft Graph (OneDrive) Refinements
- **Zero-Cache Design**:`gas-fakes` now relies entirely on the OS-level Azure CLI cache or in-memory worker identities.
- **Silent Runtime**: Once authorized, subsequent executions leverage a hardened CLI fallback to stay silent if you have an SPO license.
- **Tenant-Aware Fallback**: Implemented a strictly tenant-aware fallback for personal accounts, bypassing "SPO license" errors often encountered with business/guest identities.
- **Interactive Worker Auth**: If silent methods fail, the worker thread can now perform an interactive browser login to maintain script continuity.


### 3. Documentation & Caveats
- New dedicated [MS Graph ](../msgraph.md).
- Updated [oddities](../oddities.md) with critical details on SharePoint Online (SPO) license requirements and the current focus on Personal Microsoft Accounts.

## Action Required
If your `.env` or scripts manually set `GF_PLATFORM_AUTH` or `ScriptApp.__platform` to `workspace`, please update them to `google`. While some legacy checks remain, `google` is now the standard source of truth. You can simply rerun gas-fakes init and gas-fakes auth to update your environment and add msgraph/ksuite if you need those backends
