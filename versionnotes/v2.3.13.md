# v2.3.13

* **SpreadsheetApp (Chart Parity & Rendering)**
  * Implemented specific `EmbeddedChartBuilder` classes (`EmbeddedPieChartBuilder`, `EmbeddedBarChartBuilder`, etc.) to match Live Apps Script's strict type-casting requirements for visual formatting methods (e.g., `setTitle`, `setBackgroundColor`, `set3D`).
  * Re-architected the generic `EmbeddedChartBuilder` to dynamically compute its `toString()` value based on the assigned `ChartType`, mirroring Live GAS's underlying Java proxy behavior.
  * Added a robust **Chart Spec Translation Layer** (`chartenummapping.js`) in `FakeEmbeddedChart` to seamlessly convert Apps Script's flat builder structures into the highly-nested JSON payloads required by the Google Sheets REST API.
  * Resolved the "No basic chart type specified" API crash for `COMBO` charts by automatically defaulting unmapped data series to `COLUMN` types, matching Google's visual fallback behavior.
  * Fixed `PIE` and `HISTOGRAM` charts by automatically pivoting them out of the generic `basicChart` wrapper into their required, standalone spec configurations during the `.build()` translation phase.
  * Ensured Hex background colors are accurately parsed and nested as RGB styling objects during the API batch update process.
  * Handled the rigid Apps Script constraint where `setHiddenDimensionStrategy` hard-crashes if called on a root builder or a Pie chart builder, strictly isolating it to compatible sub-builders.

* **Documentation & Testing**
  * Added comprehensive `testsheetschart.js` suite covering all 8 standard Embedded Chart types, validating in-memory configurations and live visual grid rendering via the Sheets REST API.
  * Enforced strict Environment-Agnostic Test Design rules across `gas-fakes` agents, forbidding `isFake` assertions against internal private properties to ensure 1:1 cross-platform stability.

  * Ensured inherited properties (like `getColorType()` via `ColorBuilder` and `DataValidationBuilder` dynamic methods) are appropriately captured and analyzed in `doccreation`.

  * Handled the export limitations smoothly via `.getAs('application/pdf')` locally mapping to API exceptions for unexportable files (e.g. `text/plain`).
  * Modified the asynchronous worker interface (`workersync/synchronizer.js`) to provide proper hydration of caught server exceptions across thread boundaries, avoiding silent parsing errors or noisy stdout dumping.
  * Updated test specs (`testdrivenew.js`) to accurately reflect REST API behaviors using `.threw`.

* **General**
  * Refined `test/package.json` handling where `import "@mcpher/gas-fakes"` intelligently resolves to local files by honoring `file:..` protocol, matching authentic end-user usage in local environments.
