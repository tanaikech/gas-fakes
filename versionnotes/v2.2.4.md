# v2.2.4 Release Notes - Multi-Backend Sandbox & Resource Platform Awareness

This release introduces critical structural improvements to how `gas-fakes` handles multi-backend environments (Google Workspace, Microsoft Graph, KSuite), ensuring that sandbox tracking and cleanup operations are bulletproof when testing complex, cross-platform workflows.

## What's Changed?

### 1. Resource Platform Awareness
Resources returned by backend services (e.g. `FakeDriveMeta`) now inherently track their platform of origin (`getPlatform()`). Operations on these specific objects (like renaming, moving, or trashing) automatically trigger a targeted, temporary shift of `ScriptApp.__platform` to ensure the API call correctly routes to the originating backend regardless of the user's current global context.

### 2. Sandbox Tracking & Teardown 
The sandbox tracking arrays (`__createdIds`, `__allowedIds`) were upgraded from `Sets` to `Maps` to explicitly track the backend `platform` responsible for creating each test resource. 
During test teardown (`ScriptApp.__behavior.trash()`), the sandbox intelligently switches platforms *per file*, guaranteeing that files created in OneDrive are trashed via MS Graph, and files created in Google Drive are trashed via Google APIs, eliminating 404s and orphaned test artifacts.

### 3. Strict Default Platform Bootstrapping
Initialization was hardened to ensure that if `google` is one of the requested platforms in `GF_PLATFORM_AUTH`, it is strictly set as the primary fallback default (`ScriptApp.__platform = 'google'`) on startup, preventing unexpected behavior like MS Graph intercepting attempts to load Google Apps Script `.js` libraries via `Drive.Files.export`.

### 4. Sequential Test Suite Stability
Fixed a cross-test "hangover" bug where sequential execution in `test.js` would cause the active backend context of a failed or manually modified test (like MS Graph) to bleed into subsequent, unrelated tests (like Google Sheets testing). `testassist.js` now strictly resets `ScriptApp.__platform = 'google'` during cleanup.

### 5. Silent Microsoft Graph Replenishment
Upgraded the Microsoft Graph interactive fallback to first attempt a silent refresh by submitting a `prompt: 'none'` request through `@azure/identity`'s `InteractiveBrowserCredential`. This leverages underlying OS caches where possible and ensures a silent, headless CLI experience without unexpected browser pop-ups, only requesting manual user consent if absolutely required.

### 6. MS Graph File Overwrite Safety
Updated `OneDrive.moveFile` to explicitly append `@microsoft.graph.conflictBehavior: "replace"`. This prevents `409 NameAlreadyExists` collisions during repeated sequential test suite runs on Microsoft Graph.

### 7. Enhanced Logging
Expanded `slogger` integration across `testassist.js` and `behavior.js` to ensure the currently targeted backend platform (e.g. `on google`, `on msgraph`) is always appended to console outputs when tests are creating, whitelisting, or cleaning up resources.

### 8. Bug Fixes
- Resolved an issue in `testslidesadv.js` where the `Slides` object was incorrectly referenced as `Calendar` during dynamic builder testing.
- Removed an invalid `--client-id` parameter that caused errors in the `az account get-access-token` silent CLI fallback mechanism.
