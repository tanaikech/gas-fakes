# v2.3.3 - Dependency Simplification & WIF Auth Fix

## WIF Auth Fixes (Domain-Wide Delegation)

When running `gas-fakes` as a GitHub Action using Workload Identity Federation (WIF), the Domain-Wide Delegation (DWD) impersonation flow could crash.
- **Root Cause:** In headless WIF environments, `_getTokenInfo()` would fail when trying to look up the source identity's email from the `tokeninfo` endpoint (as WIF service account tokens often don't have this or lack the scope to hit the endpoint).
- **Fix:** `gas-fakes` now properly respects the `GOOGLE_WORKSPACE_SUBJECT` environment variable. It bypasses the `_getTokenInfo()` call entirely if the subject is already provided, preventing the crash and allowing the WIF-backed JWT signing process to succeed seamlessly.

## Dependency Simplification & Security Patching

- Replaced `archiver` and `unzipper` (and their massive nested dependency trees) with the zero-dependency `adm-zip` package.
- This successfully strips 87 dependencies from `gas-fakes` and resolves all `npm audit` warnings related to `glob`, `inflight`, and `brace-expansion` (CVE: GHSA-f886-m6hf-6m8v) that consumers were seeing when installing the package globally or as a dependency.
